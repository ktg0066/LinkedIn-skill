## [1.0.0] - 2025-11-10

### Added

#### Core Personalization System
- **PERSONALIZATION SETTINGS section** in SKILL.md
  - User profile context (name, role, organization, expertise, goals)
  - Voice & style profile (tone, content mix, writing preferences)
  - Strategic context (follower count, optimal times, collaborators)
  - Asset utilization preferences

- **Asset library framework** with complete folder structure:
  - `/examples/` - Track successful posts
  - `/templates/` - Save proven structures  
  - `/frameworks/` - Document proprietary models
  - `/case-studies/` - Store real results
  - `/voice-samples/` - Train voice matching
  - `/audience-insights/` - Track analytics
  - `/research/` - Industry data
  - `/competitors/` - Peer analysis

#### Comprehensive Templates
- **Voice samples template** - Capture authentic writing style
- **High-engagement posts tracker** - Learn from successes
- **Post templates** (4 ready-to-use):
  - Framework introduction
  - Before/after transformation
  - Contrarian take
  - Failure lesson
- **Framework documentation template** - Document methodologies
- **Case study template** - Track project results
- **Engagement patterns tracker** - Optimal times/topics/formats
- **Demographics tracker** - Audience analysis from LinkedIn Analytics

#### Documentation

## Version Numbering

We use [Semantic Versioning](https://semver.org/):

- **MAJOR** (1.x.x) - Incompatible changes requiring user action
- **MINOR** (x.1.x) - New features, backward compatible
- **PATCH** (x.x.1) - Bug fixes, backward compatible

---

## How to Update

### For Users
1. Download the latest release from [Releases](https://github.com/ktg0066/LinkedIn-skill/releases)
2. Back up your personalization data and populated assets
3. Replace the skill folder in Claude
4. Restore your personalization settings and assets
5. Review CHANGELOG for any breaking changes

### For Contributors
See [CONTRIBUTING.md](CONTRIBUTING.md) for how to propose changes.

---

## Release Schedule

- **PATCH releases** - As needed for bug fixes
- **MINOR releases** - Monthly with new features
- **MAJOR releases** - Quarterly or as needed for breaking changes

---

## Deprecation Policy

Features marked as deprecated will:
1. Be announced in CHANGELOG with **[DEPRECATED]** tag
2. Continue working for at least 2 MINOR versions
3. Show warnings when used
4. Be removed in next MAJOR version
